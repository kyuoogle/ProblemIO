    # [Problem.io](http://problem.io/) – 사용자 생성 이미지 기반 퀴즈 플랫폼
    
    사용자가 직접 이미지를 업로드하여 퀴즈를 만들고, 다른 사용자들이 그 퀴즈를 풀며 소통하는 이미지 기반 퀴즈 플랫폼입니다.
    
    짧은 단답형 정답(허용 단답)을 여러 개 등록할 수 있어, 실제 사용자의 자연스러운 입력까지 유연하게 처리할 수 있습니다.
    
    ---
    
    ## 1. 프로젝트 개요
    
    ### 1.1 목표
    
    - **이미지 기반 퀴즈 생성/풀이**가 가능한 플랫폼 구현
    - **사용자 간 소셜 인터랙션(좋아요, 댓글, 팔로우)** 제공
    - 마이페이지를 통한 **개인 활동 관리** 제공
    - 운영·확장이 쉬운 **Spring Boot + MyBatis + MySQL + Vue** 구조 설계
    
    ### 1.2 주요 특징
    
    - 이미지 업로드 기반의 퀴즈/문제 관리
    - 문제당 여러 개의 허용 단답 등록 (동의어, 말투 차이 등)
    - 좋아요, 댓글, 팔로우를 포함한 소셜 기능
    - 마이페이지: 내가 만든/푼/좋아요한 퀴즈를 한 번에 관리
    - 카카오 OAuth2 로그인 지원
    
    ---
    
    ## 2. 기술 스택
    
    ### 2.1 Backend
    
    - Java 17
    - Spring Boot (Spring MVC)
    - MyBatis
    - MySQL
    - Spring Security (JWT 기반 인증, 선택)
    - Gradle 또는 Maven (선택한 빌드 도구 명시)
    
    ### 2.2 Frontend
    
    - Vue.js
    - Vue Router
    - Axios
    
    ### 2.3 기타
    
    - Git / GitHub
    - ERD / DBML로 데이터 모델링
    - (선택) Swagger / Springdoc OpenAPI를 통한 API 문서화
    
    ---
    
    ## 3. 주요 기능
    
    ### 3.1 사용자 및 인증
    
    - 일반 회원가입 / 로그인
    - 카카오 OAuth2 로그인
    - 비밀번호 해시 저장(BCrypt)
    - 프로필 수정 (닉네임, 상태 메시지, 프로필 이미지)
    - 비밀번호 변경, 회원 탈퇴(소프트 삭제)
    
    ### 3.2 퀴즈
    
    - 퀴즈 목록 조회
        - 최신순, 인기순(좋아요 수) 정렬
        - 제목/키워드 검색
    - 퀴즈 상세 조회
        - 제목, 설명, 썸네일, 작성자, 좋아요 수, 풀이 수
        - 문제 리스트(이미지, 설명)
    - 퀴즈 생성/수정/삭제
        - 문제 이미지 업로드 경로/URL 저장
        - 문제당 여러 개의 허용 단답 등록
    
    ### 3.3 문제 풀이
    
    - 선택한 퀴즈의 문제를 순서대로 출력
    - 단답형 입력으로 풀이 제출
    - 서버에서 허용 단답 목록 기반 정답 판별
    - 제출 기록 저장
        - 정답/오답 여부, 총 문항 수, 맞힌 개수
    - 결과 화면
        - 각 문항별 정답/오답, 사용자가 입력한 값, 대표 정답 예시
    
    ### 3.4 소셜 기능
    
    - 퀴즈 좋아요 / 좋아요 취소
    - 댓글 작성 / 삭제 / 조회
    - 댓글 좋아요 (선택)
    - 유저 팔로우 / 언팔로우
    - 팔로잉 유저의 퀴즈 모아보기
    
    ### 3.5 마이페이지
    
    - 내 기본 정보 (닉네임, 프로필 사진, 상태 메시지)
    - 내가 만든 퀴즈 목록
    - 내가 좋아요한 퀴즈 목록
    - 내가 팔로잉한 유저들의 퀴즈 목록
    - 팔로워 / 팔로잉 목록
    
    ---
    
    ## 4. 시스템 아키텍처
    
    ### 4.1 전체 구조
    
    - Frontend (Vue)
    → `/api/**` 로 Backend REST API 호출
    - Backend (Spring Boot)
    → MyBatis를 통해 MySQL과 연동
    
    레이어 구조:
    
    - Controller: 요청/응답 처리, DTO 매핑
    - Service: 비즈니스 로직, 트랜잭션 관리
    - Repository(MyBatis Mapper): DB 접근
    - Domain(Entity 또는 VO): DB 모델 매핑
    
    ---
    
    ## 5. 데이터 모델 (요약)
    
    ### 5.1 주요 테이블
    
    - `users` – 사용자 정보, 소셜 로그인 정보, 프로필
    - `quizzes` – 퀴즈 기본 정보 (제목, 설명, 썸네일, 통계)
    - `questions` – 퀴즈에 속한 문제 (이미지, 설명, 순서)
    - `question_answers` – 각 문제의 허용 단답 목록
    - `submissions` – 퀴즈 단위 제출 기록
    - `submission_details` – 제출 상세 (문제 단위, 사용자 답변, 정답 여부)
    - `quiz_likes` – 퀴즈 좋아요
    - `comments` – 퀴즈 댓글
    - `comment_likes` – 댓글 좋아요
    - `follows` – 사용자 간 팔로우 관계
    
    ERD는 `/docs/erd` 또는 프로젝트 내 다이어그램 파일을 참고하도록 안내할 수 있습니다.
    
    ---
    
    ## 6. API 개요
    
    Backend는 `/api`를 prefix로 하는 REST API를 제공합니다.
    
    예시:
    
    - `POST /api/auth/signup` – 회원가입
    - `POST /api/auth/login` – 로그인(JWT 발급)
    - `GET /api/quizzes` – 퀴즈 목록 조회
    - `GET /api/quizzes/{quizId}` – 퀴즈 상세
    - `POST /api/quizzes` – 퀴즈 생성
    - `POST /api/quizzes/{quizId}/submissions` – 퀴즈 풀이 제출
    - `GET /api/users/me/summary` – 마이페이지 상단 정보
    
    자세한 내용은 `/docs/api-spec.md` 혹은 Swagger UI를 통해 확인하도록 안내할 수 있습니다.
